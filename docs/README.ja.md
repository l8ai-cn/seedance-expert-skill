## 日本語プロンプトの基本

- 利用中のプラットフォーム／画面が渡すハンドルはバイト単位で保持し、API のメディア連番は検証済みプロファイルから生成します。構造化ロールにはプロンプト用トークンを追加せず、別プラットフォームの表記を推測しません。
- 人物の同一性、衣装、構図、動きの終点を先に固定します。
- 「映画的」「エモい」「雰囲気のある」だけにとどめず、カメラ、光源、素材、空気、音に分解します。
- 最終字幕、広告コピー、法務文言は生成映像に焼き込ませず、後処理で入れます。
- 連続物語では、前の採用済みクリップの実際の終点を確認してから次のプロンプトを書きます。

## 連続クリップテンプレート

```text
物語の最終目標：[最後に到達したい状態]
採用済み事実：[前の動画で実際に起きたこと]
このクリップだけ：[今回見せる一つの可視タスク]
まだ見せないこと：[後のクリップに残す内容]
参照計画：人物画像は同一性のみ、動画はカメラのみ、音声はテンポのみを制御し、型付きバインディングは利用中のプラットフォーム用プロファイルで解決します。
プロンプト：[一つの動き + 一つのカメラ + 具体的な光 + 音]
```

## 日本語作例

型付きバインディング計画（プロンプト本文ではありません）：`product` = 商品同一性画像。レンダラーが利用中のプラットフォームに合わせて解決します。

プロンプト本文：

```text
商品のロゴ、ラベル、形状、色を正確に維持する。
変化は小さな水滴が表面を下へ流れる動きと、左から横切る柔らかい暖色光だけ。
Camera: locked product close-up, slow push-in.
Sound: quiet room tone, one small glass tick at the end.
字幕、透かし、余計な文字を追加しない。
```

## 安全な書き換え

保護されたキャラクター、実在人物、ブランド、曲、声を別の言語で隠さないでください。創作上の役割を残し、オリジナル人物、オリジナル世界、許可済み参照、または後処理案に置き換えます。
